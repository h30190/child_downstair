# 「小朋友下樓梯」同款垂直卷軸遊戲開發規格書 (Spec)

這份文件定義了「小朋友下樓梯」網頁版的開發規格。遊戲開發將使用純 HTML5, CSS3 與 Vanilla JavaScript (Canvas API)。

## 1. 遊戲核心機制 (Core Mechanics)

### 1.1 玩家控制 (Player Control)
- **左右移動**：使用鍵盤 `Left Arrow` / `Right Arrow` 或 `A` / `D` 鍵控制。
- **重力系統**：玩家角色會自動下墜，除非踩在平台上。
- **碰撞跳躍**：當玩家從上方落在平台上時，會停止下墜並與平台同步上升（畫面捲動）。

### 1.2 捲動系統 (Scrolling)
- **垂直向上捲動**：所有平台會以固定速度向上移動，模擬玩家向下移動。
- **難度遞增**：隨著時間或層數增加，捲動速度會輕微提升。

### 1.3 失敗與存活條件
- **生命值 (HP)**：
    - 初始 HP：12 點（經典版為 12 顆心）。
    - 受傷害：踩到「針刺平台」或碰到天花板針刺時扣血 (-3 HP)。
    - 生命恢復：踩到「普通平台」或「翻轉平台」可恢復少量 HP (+1 HP)，上限 12。
- **失敗條件**：
    - HP 歸零。
    - 掉出畫面底部。
    - 被推到畫面頂部撞到針刺（天花板）。

## 2. 平台類型 (Platform Types)

初步實作以下幾種平台（目前以顏色區分）：
- **普通平台 (白色/灰色)**：最基本的平台，踩踏可回血。
- **針刺平台 (紅色)**：上方帶有尖刺，踩踏會受傷扣血。
- **彈簧平台 (黃色)**：踩踏後會將玩家向上彈起。
- **傳送帶平台 (藍色)**：踩踏後會強制將玩家向左或向右傳送。
- **易碎平台 (淺藍色)**：踩踏後一段時間會消失。

## 3. 視覺設計 (Visual Design)

- **風格建議**：採用深色背景搭配細微漸層的霓虹色調色板。
- **玩家**：目前為綠色色塊。
- **天花板**：頂部固定顯示一排紅色的針刺圖形。
- **UI**：畫面上方或側邊顯示目前的層數 (Floor) 與生命值 (HP Bar)。

## 4. 技術架構

### 4.1 檔案結構
- `index.html`：遊戲容器、Canvas 元素與 UI 配置。
- `style.css`：排版與視覺美化。
- `game.js`：包含遊戲引擎、物理運算、平台生成邏輯與輸入監聽。

### 4.2 遊戲循環 (Game Loop)
使用 `requestAnimationFrame` 進行渲染，確保流暢度為 60 FPS。
